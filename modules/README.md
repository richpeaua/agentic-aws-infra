# modules/

Shared, reusable Terraform modules live here.

Create a module when the same set of resources is needed by more than one stack.
Keep single-use infrastructure inline in its stack under `stacks/<name>/`.

Prefer vetted community modules from the `terraform-aws-modules/*` registry namespace for complex or fiddly infrastructure (VPC, EKS, RDS, IAM policies).
Write raw resources for simple things.
Always pin versions.

## Interface documentation

Each module has a `README.md` with a hand-written purpose paragraph and an inputs/outputs table generated by [`terraform-docs`](https://terraform-docs.io).
The table is injected between the `<!-- BEGIN_TF_DOCS -->` and `<!-- END_TF_DOCS -->` markers; leave the markers in place and do not hand-edit the table.

- Regenerate after changing a module's variables or outputs: `scripts/gen-docs.sh`.
- CI enforces freshness: the `module-docs` job runs `scripts/gen-docs.sh --check` and fails the build if a table drifts from the code (see [`docs/ci.md`](../docs/ci.md#pipeline-anatomy)).
- A new module needs a `README.md` seeded with a purpose paragraph and the two markers before the generator can fill it; `scripts/new-stack.sh` and the `templates/stack/` scaffold produce this for you.
