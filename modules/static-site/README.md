# static-site

An intentionally-public S3 static website.
It creates a website-configured, server-side-encrypted S3 bucket plus a public-read bucket policy, so a demo site is served directly from the S3 website endpoint.
The public-access block is deliberately open and the exposure is waived in the Checkov config; the bucket name includes `account_id` so it stays globally unique across the shared account's environments.

The table below is generated by `terraform-docs` and kept fresh by CI; run `scripts/gen-docs.sh` after changing the module's variables or outputs.
